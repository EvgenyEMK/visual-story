# Two Columns (67/33)

## Overview
An asymmetric two-column layout with a two-thirds left pane (67%) and a one-third right pane (33%). Based on the classic 12-column grid system (8+4 columns), this provides generous space for primary content on the left while keeping a meaningful right column for supporting material. The right column is wide enough for substantial supplementary content — stats panels, side notes, or related items — while the left column clearly dominates. Can be used with or without a slide header.

## Visual Structure
```
┌─────────────────────────┬───────────┐
│                         │           │
│      LEFT COLUMN        │   RIGHT   │
│        (67%)            │  COLUMN   │
│                         │   (33%)   │
│                         │           │
│                         │           │
└─────────────────────────┴───────────┘
```

## When to Use
- Slides with primary content on the left and a supporting panel on the right
- Presenting main narrative alongside key takeaways, stats, or highlights
- Showing detailed content with a right-side panel for related items or links
- Content where the right column needs more space than a 25% sidebar but shouldn't compete with the main content
- Layouts where you want a clear "main + aside" visual hierarchy

## Content Recommendations
- **Left Column (67%)**:
  - Main narrative content, descriptions, or explanations
  - Primary charts, graphs, or data visualizations
  - Feature details or product specifications
  - Long-form text or article-style content
  - Images or media that need prominent display
- **Right Column (33%)**:
  - Key takeaways or highlight boxes
  - Statistics or metrics panels
  - Related items or recommendations
  - Supplementary notes or annotations
  - Quick-reference cards or callouts
  - Timeline or progress indicators
