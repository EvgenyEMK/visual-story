# Two Columns (33/67)

## Overview
An asymmetric two-column layout with a one-third left pane (33%) and a two-thirds right pane (67%). Based on the classic 12-column grid system (4+8 columns), this is one of the most widely used splits in UX design. The left column provides enough space for meaningful content — labels, short descriptions, or navigation — while the right column dominates with the primary material. Can be used with or without a slide header.

## Visual Structure
```
┌───────────┬─────────────────────────┐
│           │                         │
│   LEFT    │      RIGHT COLUMN       │
│  COLUMN   │        (67%)            │
│   (33%)   │                         │
│           │                         │
│           │                         │
└───────────┴─────────────────────────┘
```

## When to Use
- Content with a sidebar-style left panel and main content on the right
- Presenting a table of contents, chapter list, or index next to primary content
- Slides where the left column provides structural context (categories, steps, labels)
- Navigation patterns where the left third serves as a wayfinding column
- Layouts where one-quarter (25%) is too narrow but equal split (50%) gives too much space to the left

## Content Recommendations
- **Left Column (33%)**:
  - Navigation or section list
  - Category labels with brief descriptions
  - Step numbers or progress indicators
  - Legend, key, or reference panel
  - Short summaries or annotations
  - Profile info or metadata cards
- **Right Column (67%)**:
  - Main content, detailed descriptions, or explanations
  - Charts, graphs, or data visualizations
  - Images, galleries, or media content
  - Feature details or specifications
  - Interactive components or embedded content
